# 《Godot 4.4 游戏创作入门》完整课程规划文档
（草稿 v1.0，可继续迭代）

---

# 🎯 一、课程设计目标

## 1.1 总体目标
通过 10 节 × 1.5h 的课程，让 10–14 岁学生：
- 建立对 **游戏引擎是什么、怎么使用** 的整体认知
- 熟练操作 Godot 4.4 的 **场景、节点、资源、运行逻辑** 等基础概念
- 通过大量的 "任务单式学习"，建立 **明确、可执行、没有心理压力的实践路径**
- 掌握与 **AI 协作创作游戏** 的基本方法（清晰描述需求 → AI 给步骤 → 学生执行）
- 在课程结束时能够 **独立完成一款 2D 或 3D 小型游戏作品**

---

# 🧠 二、课程设计理念

## 2.1 面向零文本编程基础的学生
学生有图形化编程经验，但对文本、脚本编程不熟悉。本课程采用：
- **不直接教授 GDScript**
- **AI 协作驱动脚本生成**
- 教学生学会清晰表达需求，让 AI 生成需要的代码
- 遇到报错时教"如何向 AI 描述报错"

目标不是让学生会写代码，而是：
> **让学生会"使用游戏引擎"与"使用 AI 完成游戏逻辑"。**

---

## 2.2 强调分阶段的可控体验
课程结构采用 4 + 3 + 3：
- **4 节 2D 课程**：简单、易理解、快速上手
- **3 节 3D 课程**：新的维度、更多操作体验
- **3 节自由创作**：最重要的成就感与成品呈现

每个阶段的目标清晰、可达成，不造成学生压力。学生不会被大量理论淹没。

---

## 2.3 任务单驱动（Task Sheet Driven Learning）
- 每节课学生拿到一份「**可以直接照做**」的任务单
- 任务单格式统一：
  1. **任务目标**（学生今天能完成什么）
  2. **任务 A（必要任务）**
  3. **任务 B（扩展任务）**
  4. **和 AI 如何对话**（学生不会不知道怎么表达需求）

这种方式减少学生的焦虑，让每节课都有确定成果。

---

## 2.4 设计考虑点
课程设计时重点考虑：
1. 学生可能的挫折点（不会打字/不会找按钮/脚本报错）
2. 学生的成就点（能看到成果/能控制角色/能完成一个小回合）
3. 教学可控性（步骤可拆分、避免过度一次性传输太多任务）
4. 允许学生有不同节奏

---

# 📘 三、课程结构（4 + 3 + 3）

## **第 1–4 节：2D 游戏基础（4 节）**
核心成果：一个可玩的 2D 小游戏（移动、碰撞、收集、计分）。

## **第 5–7 节：3D 游戏基础（3 节）**
核心成果：一个小型 3D 探索体验（移动、交互、目标点）。

## **第 8–10 节：自主创作（3 节）**
核心成果：每位学生完成一个属于自己的游戏。

# 📑 四、每节课任务单（可印发）

## **Lesson 1 — 认识 Godot & 2D 场景基础**
### 🎯 今日目标
- 打开 Godot、认识界面
- 建立第一个 2D 场景
- 让角色站在地板上

### ✔️ 必做任务
1. 创建 2D 场景（Scene → New Scene → Node2D）
2. 添加角色 Sprite2D + 碰撞体（CollisionShape2D）
3. 添加地板 StaticBody2D + CollisionShape2D
4. 向 AI 描述：
   > "我在 Godot 4.4 做 2D 游戏，我有一个 Player，该怎么让它能左右移动？"

### ⭐ 扩展任务
- 给角色换图片
- 加背景图

---

## **Lesson 2 — 2D平台游戏：移动与跳跃**
### 🎯 今日目标
- 创建可控制的角色（左右移动+跳跃）
- 添加重力效果
- 创建简单平台关卡

### ✔️ 必做任务
1. 使用CharacterBody2D作为玩家节点
2. 为玩家添加移动和跳跃脚本（AI协助）
3. 创建地面和平台（StaticBody2D + CollisionShape2D）
4. 向 AI 描述：
   > "我在 Godot 4.4 中制作 2D 平台游戏，我的角色可以左右移动，但我想添加跳跃功能和重力效果，按空格键可以跳跃"

### ⭐ 扩展任务
- 添加跑步动画
- 创建多个不同高度的平台

---

## **Lesson 3 — 收集游戏：金币与计分**
### 🎯 今日目标
- 创建可收集的金币
- 实现收集机制
- 显示游戏分数

### ✔️ 必做任务
1. 创建金币场景（Area2D + Sprite2D + CollisionShape2D）
2. 实现玩家与金币的碰撞检测
3. 添加HUD显示分数
4. 向 AI 描述：
   > "我在 Godot 4.4 中制作 2D 收集游戏，当玩家碰到金币时，分数应该增加并且金币消失，怎么实现？"

### ⭐ 扩展任务
- 添加金币旋转动画
- 创建多个不同位置的金币

---

## **Lesson 4 — 敌人与生命值系统**
### 🎯 今日目标
- 添加敌人角色
- 实现简单的敌人AI
- 创建玩家生命值系统

### ✔️ 必做任务
1. 创建敌人场景（RigidBody2D + Sprite2D + CollisionShape2D）
2. 为敌人添加简单移动AI（左右移动）
3. 添加玩家生命值系统（初始3条命）
4. 向 AI 描述：
   > "我在 Godot 4.4 中制作 2D 平台游戏，我想添加敌人，当玩家碰到敌人时会失去生命值，生命值为0时游戏结束，怎么实现？"

### ⭐ 扩展任务
- 为敌人添加动画效果
- 添加游戏结束界面

---

## **Lesson 5 — 进入 3D 世界**
### 🎯 今日目标
- 认识 3D 空间界面
- 添加光源、地形
- 添加 CharacterBody3D 并能移动

### ✔️ 必做任务
1. 创建 3D 场景（新建场景 → Node3D）
2. 添加 DirectionalLight3D
3. 添加 MeshInstance3D 作为地面（CylinderMesh 或 BoxMesh）
4. 添加 CharacterBody3D 作为玩家
5. 向 AI 请求 3D 移动脚本：
   > "我在 Godot 4.4 中制作 3D 游戏，我有一个 CharacterBody3D 作为玩家，怎么让它能前后左右移动？"

---

## **Lesson 6 — 3D 交互与道具**
### 🎯 今日目标
- 添加 3D 道具（金币 Mesh）
- 让玩家捡起物体
- UI 显示分数（3D 游戏也用 2D UI）

### ✔️ 必做任务
1. 添加金币道具（MeshInstance3D + Area3D + CollisionShape3D）
2. 实现玩家与金币的碰撞检测
3. 2D UI 显示分数
4. 向 AI 描述：
   > "我在 Godot 4.4 中制作 3D 收集游戏，当玩家碰到金币时，分数应该增加并且金币消失，怎么实现？"

---

## **Lesson 7 — 做一个小迷宫 / 小岛探索**
允许学生自由选择：
- 小迷宫
- 小岛探险
- 跑酷

完成一个简单关卡。

向 AI 描述（根据选择的主题）：
> "我在 Godot 4.4 中制作 3D 迷宫游戏，我想创建墙壁围成迷宫，玩家需要找到出口，怎么实现？"
或
> "我在 Godot 4.4 中制作 3D 小岛探险游戏，我想创建一个岛屿地形，玩家可以在上面行走探索，怎么实现？"

---

## **Lesson 8–10 — 自主创作**
### Lesson 8：设计 + 框架
- 游戏主题
- 玩家 + 场景搭建
- 向 AI 描述游戏核心玩法

向 AI 描述：
> "我想在 Godot 4.4 中制作一个[游戏类型]游戏，主题是[主题]，玩家可以[玩法描述]，请帮我规划一下需要哪些节点和场景？"

### Lesson 9：结构化完善
- 实现玩法规则
- UI、场景、音效
- 调试构建

向 AI 描述：
> "我在 Godot 4.4 中制作[游戏名称]游戏，在实现[具体功能]时遇到了问题：[具体描述问题]，请帮我解决。"

### Lesson 10：打磨 + 展示会
- 加美术、音效
- 优化体验
- 作品展示

向 AI 描述：
> "我在 Godot 4.4 中完成了[游戏名称]游戏，请帮我优化[具体方面，如性能、视觉效果、用户体验等]。"
